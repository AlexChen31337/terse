### Initial Problem
User: "The Alex hub, eye web terminal failed to work, the message never got replied, plz fix"

Web terminal timing out when sending messages to alex-eye (Pi edge agent) via HTTP Chat API.

### Root Cause Investigation

#### Phase 1: MQTT Keep-Alive Issue
- Edge agent logs showed MQTT connection dropping after prompt completion
- Pi agent keep-alive: 30 seconds
- LLM processing time: 83+ seconds (Gemma 3 text encoding + tool execution)
- **Fix:** Changed keep-alive from 30s → 120s in `~/.evoclaw/agent.toml`
- Restarted Pi agent with longer timeout

#### Phase 2: MQTT Result Parsing
- Agent sending `AgentReport` structures with `report_type: "result"`
- Orchestrator only handled old-style tool results (checking for `tool` + `status` fields)
- **Fix (commit 1a16b47):** Updated `/tmp/evoclaw/internal/channels/mqtt.go`:
  - Added `AgentReport` struct matching Rust edge agent format
  - Parse `report_type` field ("result", "error", "heartbeat")
  - Route "result" reports to callback via `handleEdgeAgentResult()`
  - Extract agent ID from MQTT topic

#### Phase 3: HTTP → MQTT Routing Missing
After MQTT parsing fix, orchestrator still processed edge agents **locally** instead of forwarding to MQTT.

**Discovery:** 
- Orchestrator has `processWithAgent()` that always runs LLM locally
- No logic to detect edge vs local agents
- Agent tools work perfectly via direct MQTT (tested: got CPU temp 51.4°C)
- But HTTP Chat API → edge agent path never implemented

**User reminded me:** "It was fixed before why it came back again? Check your memory"

**Memory retrieval showed:**
- Feb 14: "HTTP Chat API doesn't support MQTT edge agents" (deemed expected behavior)
- Feb 15 (commit da9d829): MQTT tool result delivery implemented for **agentic tool loop only**
- **Gap:** Tool loop works, but prompt responses DON'T route properly

**Fix (commit d9ada22):** Implemented full HTTP → MQTT routing:
- Added `IsEdgeAgent` bool to `AgentState`
- Added `Remote` bool to `AgentDef` config
- Added `edgeResultRegistry` map for edge agent prompt results (separate from tool results)
- Implemented `processWithEdgeAgent()`:
  - Forwards HTTP requests to MQTT with unique request_id
  - Registers response channel in edgeResultRegistry
  - Waits for result with 60s timeout
  - Delivers response back to HTTP
- Updated `DeliverToolResult()` to check edgeResultRegistry first, then tool registry
- Agent initialization sets `IsEdgeAgent = def.Remote`

#### Phase 4: Payload Format Mismatch
Edge agent logs: `"missing prompt"` error

**Issue:** MQTT Send() was putting content in `payload.content`, but edge agent expects `payload.prompt`

**Fix (commit 32ee22c):** Updated MQTT channel Send():
```go
if overrideCmd == "prompt" {
    cmd.Payload = map[string]interface{}{
        "prompt":       msg.Content,
        "system_prompt": "",
        "enable_tools":  true,
    }
}
```
