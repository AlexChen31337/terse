## EvoClaw/OpenClaw Compatibility Layer

**Critical insight from Bowen:** "EvoClaw will eventually replace OpenClaw on desktop. This transition needs to be smooth."

**Problem identified:**
- Skills assumed OpenClaw-only environment (cron jobs, sessionTarget, etc.)
- EvoClaw has built-in tiered memory system (hot/warm/cold + consolidator)
- Duplication: skill scripts vs EvoClaw native memory
- Skills wouldn't work when EvoClaw replaces OpenClaw

**Solution implemented:**
Added `evoclaw memory` CLI commands that delegate to built-in memory system:
- `evoclaw memory consolidate --mode quick|daily|monthly`
- `evoclaw memory store --text "fact" --category "category"`
- `evoclaw memory retrieve --query "search"`
- `evoclaw memory status`

**Dual-mode wrapper:**
```bash
# skills/tiered-memory/scripts/memory
if command -v evoclaw &> /dev/null; then
    exec evoclaw memory "$@"  # EvoClaw environment
else
    exec python3 memory_cli.py "$@"  # OpenClaw environment
fi
```

**Changes:**
- cmd/evoclaw/main.go: Add 'memory' subcommand
- internal/cli/memory.go: New CLI handler (285 lines)
- internal/memory/manager.go: Add GetConsolidator(), Store(), Search(), GetStatus()
- skills/tiered-memory/scripts/memory: Environment detection wrapper
- skills/tiered-memory/install.sh: Document dual-mode operation
