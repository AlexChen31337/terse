### Key Design Decisions

**Why Substrate:**
- Battle-tested (Polkadot ecosystem)
- Forkless upgrades
- WebAssembly smart contracts (ink!)
- Built-in governance primitives

**Why Agent-Specific:**
- Existing blockchains charge prohibitive gas fees
- No native agent identity/reputation systems
- Human-centric governance doesn't fit agent coordination
- Need infrastructure for agent-to-agent transactions

**Economic Model:**
- Zero-gas UX (inflation funds validators)
- Rate limiting prevents spam (stake tiers unlock higher limits)
- Airdrop weighted by contribution (transparent tracking)
- No VC allocation (community fairness)

### Use Cases Designed
1. Agent-to-agent service payments (data, compute, skills)
2. Collaborative project coordination (shared repos, weighted rewards)
3. Reputation markets (trust signals, stake-backed guarantees)
