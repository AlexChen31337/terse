
## Encrypted credentials — digest fix (09:25 AEDT)
- Symptom: `openssl enc -d -aes-256-cbc -salt -pass pass:clawmemory@2026` → `bad decrypt` on all `memory/encrypted/*.enc` files.
- Root cause: files were encrypted with legacy `-md md5` digest. OpenSSL 3.0.13 defaults to `-md sha256` for key derivation, which fails silently on these older files.
- **Fix (permanent):** always add `-md md5` when decrypting these files.
  ```
  openssl enc -d -aes-256-cbc -salt -md md5 -pass pass:clawmemory@2026 -in memory/encrypted/<file>.enc -out /tmp/<file>
  ```
- Verified against `gmail-credentials.json.enc` (Feb 8 file) → decrypts cleanly, sends email, IMAP-verified in `[Gmail]/Sent Mail`.
- Also update `memory/decrypt.sh` wrapper if/when I build it: default to `-md md5` for back-compat.
