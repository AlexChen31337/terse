## Pre-Compaction Flush #3 — 21:17 AEST

### XML Leakage Fix v2 (claude-code-cli provider)

**Problem:** The `claude-code-cli` plugin at `/home/bowen/.openclaw/extensions/claude-code/dist/server.js` was leaking XML tool call markup into chat responses. Two distinct issues:

1. **CLI error passthrough** (fix v1, applied ~20:46): Error messages like "Invalid API key · Fix external API key" from Claude CLI workers were being returned as HTTP 200 content instead of HTTP 500 errors. Added `isCliError()` pattern matching to detect known CLI errors and return HTTP 500 so the gateway triggers fallback.

2. **XML content leaking alongside tool calls** (fix v2, applied ~21:10): After extracting XML tool calls (`<function_calls>`, `<tool_call>`, `<invoke>`), the `extractToolCalls()` function left XML fragments in the `content` field. Evidence from logs: `finish=tool_calls tool_calls=1 content_len=2546 raw_has_xml=true` — tool calls were correctly extracted but 2546 chars of XML-containing content was still sent to chat.

**Fix v2 details:**
- Added aggressive XML tag stripping after tool call extraction: strips `</?invoke>`, `</?parameter>`, `</?function_calls>`, `</?tool_call>`, `</?antml:invoke>`, `</?antml:parameter>` and their attribute variants from leftover content
- If content becomes empty after stripping, set to `null`
- Applied to both non-streaming `toOAIResponse()` and streaming `extractToolCalls()` paths
- Also fixed Format 3 regex: `<invoke>` opening → `</invoke>` closing mismatch, now handles both variants

**Key files:**
- Plugin source: `/home/bowen/.openclaw/extensions/claude-code/dist/server.js`
- Plugin config: `/home/bowen/.openclaw/extensions/claude-code/openclaw.plugin.json`
- Plugin ports: 8090, 8091 (verified healthy)
- Gateway restarted after each fix round

**Architecture notes for future reference:**
- Plugin spawns `claude --bare --stream-json` worker processes in a `PersistentPool` (POOL_SIZE=2 per model, MAX_REQUESTS=100 before worker replacement)
- Models pooled: `claude-haiku-4-5`, `claude-sonnet-4-6`, `claude-opus-4-6`
- Reads OAuth token from `~/.claude/.credentials.json` (`claudeAiOauth.accessToken`)
- Gateway sends OAI-format requests to plugin; plugin injects tools as XML in system prompt, model responds with XML tool calls, plugin parses XML back to OAI `tool_calls` format
- When `supportsTools` is not set on the provider, the gateway ALSO injects tool XML — potential for double injection (needs monitoring)
