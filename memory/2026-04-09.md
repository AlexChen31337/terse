## claude-direct Plugin Rewrite (13:34–14:03 AEDT)

### Root Cause Found
- Plugin at https://github.com/AlexChen31337/openclaw-claude-direct was text-injecting tools into system prompt instead of using native Anthropic tool API
- `convertMessages()` flattened all content to plain text, dropped `tool` role messages, dropped images/cache_control
- Model couldn't actually call tools — it could only output XML that the plugin regex-parsed

### Fix Applied (v2.0.0 → thin relay)
- Rewrote server.ts as a thin relay: inject auth headers + billing, relay everything else
- Native Anthropic tools: OAI `tools[]` → Anthropic `tools[]` with `input_schema`
- Full message conversion: `tool_calls` ↔ `tool_use`, `tool` role → `tool_result`
- Preserves cache_control blocks, multi-modal content, images
- Proper streaming with `content_block_start`/`input_json_delta` for tool calls
- 655 lines → 337 lines
