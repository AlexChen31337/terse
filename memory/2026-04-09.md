## Rate Limit Fix & Anthropic Messages Endpoint (14:45–15:33 AEDT)

### Bug: Rate limit → HTTP 500
- **Root cause:** Anthropic API error object lacks `.status` field. Plugin code `resp.error?.status || 500` always returned 500 for ALL errors including 429 rate limits
- **Effect:** When claude-direct-1 (token-1) was rate-limited, gateway got 500 instead of 429 → gave up entirely instead of falling back → silence, no response
- **Fix:** Infer HTTP status from `resp.error.type`: rate→429, auth/permission→401, not_found/model→404, else→500
- **Commit:** `d5abbea` pushed to GitHub

### Fallback Chain Config
- Token-0 (claude-direct, port 8092) tried first
- Token-1 (claude-direct-1, port 8093) as fallback
