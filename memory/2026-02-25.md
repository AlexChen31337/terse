### ADR-007: Native OpenClaw Memory Lifecycle

**Key decision:** Replace skill-based memory (tiered-memory, hybrid-memory, session-guard) with native OpenClaw config primitives. NOT custom plugins — OpenClaw already ships everything needed.

**OpenClaw native capabilities confirmed:**
- `memorySearch`: SQLite + sqlite-vec (bundled at `openclaw/node_modules/sqlite-vec-linux-x64/vec0.so`), hybrid BM25+vector, `onSessionStart` auto-inject, file watcher, intervalMinutes reindex
- `compaction.memoryFlush`: auto-saves context before compaction hits token limit
- `contextPruning`: cache-ttl mode, prunes old tool results automatically

**Config patches applied (3x via `config.patch`, NOT `config.apply`):**
1. `memorySearch` enabled — SQLite vector, hybrid query, onSessionStart, watch, 30min interval
2. `compaction.memoryFlush` — safeguard mode, fires at 150k tokens
3. `contextPruning` — ttl 2h, keepLastAssistants 10, deny Read+exec tool results

**Migration Builder running as of session end:**
- Archiving: tiered-memory, hybrid-memory, session-guard → `skills/archived/`
- Editing: AGENTS.md (remove manual retrieval protocol), HEARTBEAT.md (remove session detection + hybrid reindex), SOUL.md (replace "Search First" with "Memory is automatic now")
- Rollback: single config.patch to disable + mv skills back + git checkout files

**ADR-007 committed:** `docs/architecture/ADR-007-native-memory-lifecycle.md`

**Important nuance — native vs Python tiered-memory:**
| | Native OpenClaw | Python tiered-memory |
|---|---|---|
| Retrieval | Vector + BM25 hybrid, fast, free | LLM-powered tree navigation, slow, costly |
| Reliability | Always-on, no agent cooperation | Agent must remember to call |
| Tiers | Flat index | Warm/cold tree hierarchy |
| For ambiguous queries | OK | Better (reasoning path) |
Trade-off accepted: reliability > sophistication.

- **PBR quality over speed** — Bowen explicitly asked to use orchestrator after initial direct-builder approach. Review phase caught real bugs (setInterval leak, CSWSH, personal info in docs, missing test coverage).
- **Reviewer fixed real issues** — every Review phase found something non-trivial. Don't skip it.
