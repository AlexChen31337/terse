### GPU0: Qwen3.5-35B Context Upgrade ✅
- **Before:** 8K ctx, 4 slots, q8_0 KV cache
- **After:** 262K ctx, 2 slots (131K each), q4_0 KV cache
- **Key insight:** Gated Delta Net architecture — only 10/50 layers are attention, rest is recurrent (fixed-size buffer)
- **Service:** `/etc/systemd/system/llama-server-3090.service` on GPU server, enabled on boot
- **VRAM:** 18.2 GB / 24.6 GB used
- **CRASH NOTE:** First attempt with `-np 4` (4 slots × 262K) OOM'd the 16GB RAM machine. Had to recover from kernel OOM.

### Model Comparison: Qwen3.5-35B vs Gemma 4 26B
- **Verdict:** Qwen3.5-35B wins for our use case
- Active params: 3B vs 4B — close but Qwen3.5 architecture stronger
- Qwen handles aggressive quantisation better (MoE sparse routing dilutes errors)
- Suggested: put Gemma 4 on GPU1 (3080) if experimenting

### ⚡ BREAKTHROUGH: OAuth Tokens Work Directly on Messages API
- **Tested at 16:05 AEDT:** `curl -H "x-api-key: $OAUTH_TOKEN" -H "anthropic-version: 2023-06-01"` → HTTP 200
- **No impersonation headers needed!** Anthropic changed their policy since morning session
- Haiku 4.5: works (HTTP 200). Sonnet 4.6: 429 rate limit (but auth accepted, not rejected)
- **This means:** proxy-0/proxy-1 should work directly without the shim

### OpenClaw Plugin Built: `openclaw-plugin-claude-code` ✅
- **Repo:** https://github.com/AlexChen31337/openclaw-plugin-claude-code
- **Plugin id:** `claude-code`
- **Provider:** `claude-code` with `anthropic-messages` transport
- **Models:** claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5 (all $0 cost, 200K ctx)

- **Status:** Built, compiled, pushed to GitHub. NOT published to npm yet.

- **BUT:** OAuth tokens now accepted on direct API (tested and confirmed)

## Claude Code Plugin Refactor (20:40)

**Problem:** Old Python shim on port 8090 was batch-only, no streaming, no proper OpenAI compatibility.

**Solution:** Replaced with Node.js TypeScript streaming proxy at `/tmp/openclaw-plugin-claude-code/`.

**Key findings from today's investigation:**
- Anthropic gates Sonnet/Opus for OAuth tokens at server level (NOT just headers)
- Captured full CC CLI headers via mitmproxy — replicating them via curl still gets 429
- Only the actual `claude` binary can access Sonnet/Opus with subscription OAuth tokens
- Direct API with same token = Haiku only (429 on Sonnet/Opus)

**New architecture:**
- `/tmp/openclaw-plugin-claude-code/dist/server.js` on port 8090
- systemd user service: `claude-code-shim.service`
- Supports both streaming (SSE via `claude --output-format stream-json`) and batch
- OpenAI-compatible `/v1/chat/completions` endpoint
- $0 cost — uses Claude Code Pro subscription

**Old files archived:**
- `scripts/claude_code_shim.py` → `scripts/claude_code_shim.py.bak`
- Old systemd service file backed up at `claude-code-shim.service.bak-20260405-183919`
