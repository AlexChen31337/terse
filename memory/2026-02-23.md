## EvoClaw Tool Loop Fix (Morning)
**Bug:** Tool loop in `internal/orchestrator/toolloop.go` made a redundant final LLM call after the loop completed. When the LLM returned no tool calls (final text answer), that content was appended to messages, then ANOTHER LLM call was made. The second LLM saw conversation ending with its own assistant message and returned empty content — hence "empty response in API JSON".

**Root cause:** The `Execute()` method always made a final `callLLM()` after the loop, even when the loop already produced a text-only response. This second call was:
1. Wasteful (extra API call + tokens)
2. Buggy — LLM sees its own response as last message → returns empty

**Fix:** Track `finalContent` from the loop. Only make a summary LLM call when:
- Loop hit max iterations (last messages are tool results, needs summarisation)
- No text response was ever produced (edge case)
