## SKILLRL Research (2026-02-28)
- **arXiv:** https://arxiv.org/abs/2602.08234
- **Paper:** "SKILLRL: Evolving Agents via Recursive Skill-Augmented Reinforcement Learning"
- **Authors:** Peng Xia (UNC Chapel Hill), Jianwen Chen, Hanyang Wang, +11 others
- **Code:** https://github.com/aiming-lab/SkillRL
- **Full report:** `/home/bowen/.openclaw/workspace-foundry/research/skillrl-integration.md`

**Key findings:**
- SKILLRL adds **skill learning from successes** to complement EvoClaw's failure-based RSI
- EvoClaw is missing: skill abstraction layer, LLM-powered distillation, embedding-based retrieval, context injection
- Proposed package: `internal/skillbank/` with clean Go interfaces (Store, Distiller, Retriever, Updater, Injector)
- **Implementation:** 3 phases, ~6-8 weeks total
  - Phase 1: Foundation (skill types + JSONL store) — can start now
  - Phase 2: LLM distillation + retrieval
  - Phase 3: Recursive evolution
- **Priority:** HIGH — could be EvoClaw's Phase 3 killer feature

**What SKILLRL provides that EvoClaw lacks:**
1. Hierarchical skill bank (general_skills + task_specific_skills + common_mistakes)
2. LLM-powered skill distillation from trajectories (not just rule-based pattern detection)
3. Embedding-based adaptive retrieval (Qwen3-Embedding-0.6B)
4. Recursive co-evolution (skill library + policy evolve together)
5. Context injection (skills prepended to agent prompts)

**RSI Loop vs SKILLRL — Complementary, Not Competitive (2026-02-28):**
- **RSI loop (OpenClaw):** Logs failures → generates improvement proposals → fixes mistakes
- **SKILLRL (EvoClaw):** Learns from ALL outcomes (success + failure) → reinforcement learning → skill selection
- **Different layers:**
  - RSI: Python skill, session-level, failure-focused, rich context (files/memory/tools)
  - SKILLRL: Go package, agent-level, success+failure, stateless skill selection
- **Integration path:** Port SKILLRL to EvoClaw first (better fit), then backport "success learning" to OpenClaw's RSI loop as lightweight enhancement
- **No RSI loop update needed right now** — already working well for OpenClaw's use case
