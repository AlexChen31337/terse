### Design Decisions Made
1. **Persona files are per-agent-type, not universal**
   - Trading agents: genome.toml + style.toml (no personality needed)
   - Companion agents: character.toml + personality.md + boundaries.md + memory.json
   - The persona layer is a capability module — load when needed

2. **Orchestrator is the translator layer**
   - Edge agents speak structured data (JSON, MQTT)
   - Orchestrator translates to natural language for human channels
   - Only orchestrator needs voice/personality prompts

3. **Trader personality = trading genome**
   - Each human's trading style encoded in style.toml
   - Evolution engine optimizes WITHIN personality bounds
   - Hard constraints (no falling knives) are never evolved
   - Agent becomes "better version of your trading self"

4. **Companion device use cases** (not just toys!)
   - Kids: educational companion
   - Adults: stress relief, loneliness
   - Seniors: aged care, medication reminders, cognitive health, dignity
   - Cloud STT/TTS (not local) — latency acceptable, keeps agent tiny

5. **Robot use case PARKED** — wait for spatial models

6. **OS strategy: phased approach**
   - Phase 1 (now): Run on Linux/RTOS/bare metal
   - Phase 2 (6mo): ClawOS Lite (minimal Linux distro, boots into EvoClaw)
   - Phase 3 (12-18mo): Design ClawOS kernel spec
   - Phase 4 (24mo+): Build ClawOS if community supports it
   - Philosophy: "Ship on Linux now, earn the right to build ClawOS later"

7. **Test coverage bar**: 90% hard minimum, 95% nice-to-have
   - Accepted 88.3% Go overall since all business logic packages individually 90%+
