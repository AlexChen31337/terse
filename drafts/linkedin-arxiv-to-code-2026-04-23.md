Launching Arxiv-to-code — an open-source GitHub org where AI coding agents replicate arxiv papers daily.

The problem: research moves faster than reference implementations. A paper drops, and working code lags by months — if it shows up at all.

The experiment: a pipeline that picks high-impact papers, reads them end-to-end with Claude Sonnet 4.6, plans a minimal viable replication, scaffolds a repo, writes tests, verifies, and publishes under MIT license.

So far: 18 repos shipped across multimodal reasoning, RL fine-tuning, agent orchestration, and security protocols.

This is not polish-grade research code. It's a working baseline — enough to reproduce the core claim or iterate from. The bar is: clone, run, read, extend.

Why it matters: AI agents shouldn't just write your crud API. They should close the loop between published ideas and runnable artifacts, at a cadence humans can't sustain alone.

If you have a paper you want replicated, or want to contribute a review pass, drop a comment.
